# Access-control matrix

| Capability | Customer | Support | Fraud/compliance | Finance | Admin |
|---|---:|---:|---:|---:|---:|
| Own accounts/payments | Yes | Masked read | Case read | Reconciliation read | No default |
| Other customer data | No | Need-to-know | Assigned cases | Financial evidence | No default |
| Freeze/dispute | Own request | Initiate | Initiate | No | No |
| Financial repair | No | Request | No | Approve/execute with dual control | No unilateral |
| Biometric evidence | Own decision only | No | Risk decision only | No | No |

Every object operation rechecks tenancy; roles alone are insufficient.
